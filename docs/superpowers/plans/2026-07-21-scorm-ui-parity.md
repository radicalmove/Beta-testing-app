# SCORM UI Parity Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make SCORM marker placement, composer typography, toolbar state, and anchored-comment scrolling match the established Moodle behaviour.

**Architecture:** Retain the existing parent-toolbar/elected-worker structure. Share the placement cursor, publish embedded placement intent to the overlay, and make renderer visibility depend on the recovered anchor being inside the current embedded viewport.

**Tech Stack:** TypeScript, Chrome Manifest V3 content scripts, Shadow DOM, Happy DOM, Node test runner.
