# Beta Testing App 0.5.0 Stabilisation Implementation Plan

> **For Codex:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Resolve every known 0.4.69 pilot issue, preserve confirmed working Moodle and SCORM behaviour, and produce a verified 0.5.0 build ready for focused manual retesting.

**Architecture:** Keep the FastAPI server authoritative for identity, permissions, visibility, sharing, status, and attachments. In the extension, derive every list, marker, filter, counter, and navigation action from one viewer-specific comment projection. Treat Rise navigation as asynchronous best-effort recovery with an explicit in-app fallback.

**Tech stack:** Python/FastAPI/SQLAlchemy/Pytest; TypeScript/Chrome Manifest V3/Vitest/JSDOM; HTML/CSS/Jinja.
